# 13 挂库到现有工程

> **本文职责**：说明如何把 Bmelod 挂入已有 Cube / SDK / CMake 工程（类 FreeRTOS 两步模型）。  
> **不负责**：Port 里具体实现哪些 `bm_drv_*` 符号 → [03-Port移植层](03-Port移植层bm_port.md)；HAL 契约与混合域绑定 → [01-HAL契约与移植要点](01-HAL契约与移植要点.md)。

## 1. 目录与类比

Bmelod 目录布局类比 **FreeRTOS**：

```text
Source/          库内核（类比 FreeRTOS/Source/）
include/         公共头文件
portable/        平台相关（类比 FreeRTOS/portable/）
Demo/            示例（类比 FreeRTOS/Demo/）
```

| FreeRTOS | Bmelod |
|----------|--------|
| `Source/*.c` | `Source/core/`、`Source/hal/`、`Source/hybrid/` |
| `FreeRTOSConfig.h` | `bm_config.h` |
| `portable/.../port.c` | [`portable/template/bm_port.c`](../portable/template/bm_port.c) |
| 源码 / 预编译库 | 方式 A 源码 / 方式 B 静态库 |

## 2. 两步集成（顺序固定）

```text
① 移植（Port）     复制 portable/template/bm_port.c，对接 Cube / SDK
② 库集成           Source/ 源码进工程，或链接 libbm_*.a
```

```text
┌─────────────────────────────────────────┐
│  你的应用（main、业务、厂商 startup/HAL）   │
├─────────────────────────────────────────┤
│  Port（bm_port.c，自 template 复制）      │
├─────────────────────────────────────────┤
│  Bmelod 库（Source/ + include/）         │
├─────────────────────────────────────────┤
│  bm_config.h（类比 FreeRTOSConfig.h）    │
└─────────────────────────────────────────┘
```

| 步骤 | 内容 | 文档 |
|------|------|------|
| ① 移植 | 实现 `bm_drv_*_api` | [03-Port移植层](03-Port移植层bm_port.md) |
| ② 集成 | 源码或静态库 | 本文 §3–§4 |

## 2.1 头文件与 API 入口

所有公共头文件在 **`include/` 根目录**（无子目录）。Include Path 只需：

1. 应用目录（`bm_config.h`）
2. `bmelod-baremetal/include/`

```c
#include "bmelod.h"         /* 按 BM_CONFIG_ENABLE_* 暴露子系统 */
#include "bm_hal_uart.h"    /* 板级 HAL 按需 */
```

CMake 会将 `BM_ENABLE_*` 同步为 `BM_CONFIG_ENABLE_*`。详见 [../02-构建与工具链/03-bmelod头文件与include](../02-构建与工具链/03-bmelod头文件与include.md)。

## 3. 方式 A — 源码集成

列出库源文件（**不含** Port）：

```bash
python tools/list_sources.py --profile event --format keil
python tools/list_sources.py --profile event --list-includes --format keil
```

在应用工程中**另加**自 template 复制的 `bm_port.c`。

**CMake 最小片段：**

```cmake
include(.../cmake/bmelod.cmake)
bmelod_configure(ROOT ... PROFILE event BACKEND external CONFIG bm_config.h)
target_sources(app PRIVATE Core/Src/bm_port.c)
bmelod_link(app)
```

完整片段：[`cmake/integration-snippet/CMakeLists.append.txt`](../cmake/integration-snippet/CMakeLists.append.txt)

## 4. 方式 B — 静态库

预编译 `libbm_*.a` 后链接；**Port 仍须在应用工程编译**。详见 [../02-构建与工具链/02-预编译静态库](../02-构建与工具链/02-预编译静态库.md)。

## 5. PROFILE 裁剪

| PROFILE | 内容 |
|---------|------|
| `minimal` | 事件、内存池、日志 |
| `event` | + 模块、看门狗 |
| `servo` | + HRT、ticker、exec |
| `full` | 全开（含 stream、pipeline、algorithm） |

与 [../02-构建与工具链/01-CMake选项与bm_config](../02-构建与工具链/01-CMake选项与bm_config.md) 中 `BM_ENABLE_*` 选项一致。
