## 1. 术语对照

文档与代码里「HRT」一词有两层含义，容易混淆：

| 叫法 | 代码标识 | 触发源 | 执行上下文 |
|------|----------|--------|------------|
| **Hardware HRT**（常简称 HSRT） | `BM_EXEC_SLOT_HARDWARE` | PWM 更新、ADC 完成、比较器 IRQ | HAL 在中断里**直接**调回调 |
| **Scheduled HRT** | `BM_EXEC_SLOT_PERIODIC` → `bm_hrt` | 专用定时器 tick（如 TIM6） | `bm_hrt` ISR 扫描到期槽 |
| **SRT** | 主循环 | 无硬件截止 | `bm_event_process` / `bm_ticker_poll` |

**关键实现事实**（读源码时务必记住）：

- `bm_hrt` **只分发** `BM_HRT_TRIGGER_TIMER` 的槽；`bm_hrt.c` 中 `hrt_dispatch()` 会跳过其他触发类型。
- **Hardware HRT 不经过 `bm_hrt`**。`bm_exec_init_all()` 通过 `bind()` 把 HAL IRQ 绑到 `bm_exec_run_binding()`，再由它调用你的 `run` 函数。
- `bm_exec` 不保存 PWM/ADC 等外设触发枚举；Hardware 路径由 HAL `bind` 适配器决定事件源，`bm_hrt` 内部周期槽只使用 `TIMER`。

```text
                    ┌──────────────────────────────────────┐
  ADC/PWM IRQ ─────►│ Hardware HRT                         │
                    │  HAL → bm_exec_run_binding → step    │
                    └──────────────────────────────────────┘

  TIM6 tick   ─────►│ Scheduled HRT (bm_hrt)               │
                    │  hrt_dispatch → callback → step      │
                    └──────────────────────────────────────┘

  main loop   ─────►│ SRT                                  │
                    │  bm_ticker_poll / bm_event_process   │
                    └──────────────────────────────────────┘
```

## 2. 域边界与禁止事项

| 规则 | 说明 |
|------|------|
| HRT ISR 禁止 | `bm_event_process`、订阅回调、`bm_mempool_alloc`（常规路径）、`bm_channel`、`BM_LOG` |
| HRT ISR 允许 | 读 ADC、写 PWM/比较器、写 `bm_snapshot`、调 `bm_event_publish_copy_from_isr`（仅入队，不执行回调） |
| SRT 允许 | 事件、池、通道、Shell、看门狗、读 `bm_snapshot` |
| 亚微秒保护 | 比较器、定时器 Break 等硬件路径，不依赖 CPU 调度 |

`bm_critical` 保护的是 **SRT 共享结构**。HRT 路径不应长时间持锁；分析 WCET 时须计入 Hardware HRT 对 Scheduled HRT 的抢占。

跨域数据选型见 [§7](#7-跨域通讯)。违反域边界时的排障见 [../04-测试与排障/03-运行时约束与排障](../04-测试与排障/03-运行时约束与排障.md)。

## 3. Hardware HRT 用法

### 3.1 直接 HAL 绑定（不用 `bm_exec`）

适合单环、资源极简的场景。

#### 绑定物：`bm_hal_hrt_binding_t`

```c
typedef struct bm_hal_hrt_binding {
    void (*callback)(void *context);   /* 环算法入口 */
    void *context;                     /* 传给 callback 的用户状态 */
} bm_hal_hrt_binding_t;
```

**应用侧**

```c
static void current_loop_step(void *ctx) {
    motor_state_t *m = (motor_state_t *)ctx;
    uint16_t adc;
    bm_hal_adc_read_injected(&BM_HAL_ADC1, 0, &adc);
    m->duty = pi_update(&m->pi, adc);
    bm_hal_pwm_set_duty(&BM_HAL_PWM_TIM1, 0, m->duty);
}

void motor_hrt_bind(void) {
    bm_hal_hrt_binding_t binding = {
        .callback = current_loop_step,
        .context  = &motor_state,
    };
    bm_hal_adc_bind_complete(&BM_HAL_ADC1, &binding);
}
```

**时序**：`bind_complete` 保存 callback → 硬件 JEOC → 向量 ISR 调用 `callback(context)`。

仿真环境调用 `bm_hal_adc_sim_fire_complete()` 等价于硬件 IRQ。

- `bm_hal_adc_bind_complete(adc, NULL)`：关中断并清空绑定。
- `bind_complete` **不**初始化 ADC 时钟、注入序列、PWM 同步触发。
- 环函数在 IRQ 上下文执行，禁止 `BM_LOG`、`bm_event_process` 等 SRT 操作。

更多 HAL 契约见 [../03-移植与IDE集成/01-HAL契约与移植要点](../03-移植与IDE集成/01-HAL契约与移植要点.md)。

### 3.2 通过 `bm_exec`（推荐，多实例）

与 §3.1 **中断对接方式相同**，差别是框架把 `callback` 固定为 `bm_exec_run_binding`，再转调 `slot->run`：

```c
static const bm_exec_slot_t g_servo_slots[] = {
    {
        BM_EXEC_SLOT_HARDWARE,
        0u,                              /* Hardware 槽必须为 0 */
        current_step,
        bind_adc,
        "current"
    },
};

static int bind_adc(const bm_exec_t *inst,
                    const bm_hal_hrt_binding_t *binding) {
    (void)inst;
    return bm_hal_adc_bind_complete(&BM_HAL_ADC0, binding);
}
```

组装全流程见 [§10](#10-bm_exec-组装流程)。

## 4. Scheduled HRT（`bm_hrt`）

### 4.1 接线

```c
static void speed_loop(void *ctx) {
    axis_t *a = (axis_t *)ctx;
    /* 速度环 */
}

static const bm_hrt_slot_t slots[] = {
    { .period_us = 1000, .trigger = BM_HRT_TRIGGER_TIMER,
      .callback = speed_loop, .context = &axis, .name = "speed" },
};
bm_hrt_init(slots, 1);
bm_hrt_start();
```

```text
bm_hrt_init → g_slots[]
bm_hrt_start → bm_hal_timer_set_callback(hrt_timer_isr)
TIM IRQ → hrt_timer_isr → hrt_dispatch → speed_loop(context)
```

### 4.2 约束

- `period_us` 须整除 `BM_CONFIG_HRT_TICK_US`，换算后不超过 `INT32_MAX` tick。
- 槽数 ≤ `BM_CONFIG_HRT_MAX_SLOTS`。
- 错过 deadline 时递增 `bm_hrt_get_deadline_missed()`；钩子默认空实现，覆盖实现不得在 ISR 中打日志。

### 4.3 经 `bm_exec`

`BM_EXEC_SLOT_PERIODIC` 槽在 `init_all` 时并入 `g_hrt_slots[]`。`bm_exec_start_all()` 在**全部**实例 `ops->start` 成功后启动 HRT；Periodic 与 Hardware `run` 都仅在会话 `STARTED` 后执行。

## 6. 同步域 `bm_sync`

同步域解决 **多路 PWM/ADC 的硬件时序对齐**（多轴同时起步、交错并联相位差等）。**不负责数据通讯**。

### 6.1 描述符与状态机

```c
typedef struct {
    const char *name;
    const bm_hal_timer_t *master_timer;
    const bm_exec_t *const *members;
    const uint32_t *phase_ticks;
    uint32_t member_count;
} bm_sync_domain_t;
```

```text
configure → HAL 配主从定时器/相位，不输出
arm       → 武装，仍不启动
trigger   → HAL 按 phase_ticks 同时释放各成员 PWM/定时器（one-shot，须重新 arm）
safe_stop → 停机并清状态
```

**`trigger` 不直接调用你的 `step` 函数**。各轴 `axis_step` 挂在 `BM_EXEC_SLOT_PERIODIC`；`sync_trigger` 让各轴 PWM 在同一硬件相位开跑。

### 6.2 推荐调用顺序

```c
bm_exec_init_all(instances, count);
bm_sync_configure(&domain);
bm_sync_arm(&domain);
bm_exec_start_all(instances, count); /* 内部在全部 start 后再 bm_hrt_start */
bm_sync_trigger(&domain);
```

未链平台 sync HAL 时 `configure` 失败；PC 须链 `bm_sync_hal_native.c`。

## 7. 跨域通讯

### 7.1 选型矩阵

| 方向 | 传什么 | 推荐机制 | 不推荐 |
|------|--------|----------|--------|
| HRT → SRT | 最新测量值、状态字 | `BM_SNAPSHOT_PUBLISH` / `READ` | `bm_event` 高频捎带 |
| HRT(快) → HRT(慢) | 降采样结果 | 同上 `bm_snapshot` | 共享无保护全局变量 |
| ISR → SRT | 「发生了」+ 小 payload | `bm_event_publish_copy_from_isr` | 在 ISR 里调订阅者 |
| SRT → SRT | 业务消息 | `bm_event` / `bm_channel` | — |
| SRT → HRT | 设定值、使能位 | 反向 snapshot 或单写者 `volatile` 字段 | SRT 持锁写复杂结构 |

**原则**：事件是 **信号**（通知刷新），不是 **数据管道**。实时流走 snapshot。

### 7.2 `bm_snapshot` 三缓冲

```c
BM_SNAPSHOT_DEFINE(pack_current, float);
static pack_current_snapshot_t g_pack = BM_SNAPSHOT_INITIALIZER;

/* 写者（HRT） */
BM_SNAPSHOT_PUBLISH(g_pack, &amps);

/* 读者（SRT 或慢 HRT） */
BM_SNAPSHOT_READ(g_pack, &amps);
```

- 无锁，适合单写者单读者；
- 允许丢中间帧，读者始终拿到 **最新完成的一帧**。

### 7.3 SRT → HRT 反向路径

1. **反向 snapshot**：SRT `PUBLISH` 设定值，HRT `READ`（HRT 读路径须极简）。
2. **单写者约定**：SRT 写 `volatile` 标量，HRT 只读；启用前用 `bm_memory_barrier_release()`。
3. **使能/模式**：通过 `bm_exec` 的 `ops->start` / `safe_stop` 在 SRT 侧切换。

示例对照见 [02-main §6](02-main骨架与数据流.md#6-示例数据流速查)。

## 8. Block/Frame 槽与流式 drain

| kind | 触发 | 数据 |
|------|------|------|
| `BM_EXEC_SLOT_BLOCK` | 主循环 `bm_stream_drain_on_this_cpu` / `bm_exec_drain_streams` | `bm_stream` 零拷贝块 |
| `BM_EXEC_SLOT_FRAME` | 同上（帧粒度元数据） | 行/帧缓冲 |

单核主循环在 `bm_event_process` 前调用 `bm_exec_drain_streams(budget)`。

## 9. 推荐初始化顺序（Control 应用）

```text
1. bm_event_reset()
2. bm_module_init_all() / start_all()
3. bm_exec_init_all(instances, count)     /* 组 HRT 表 + Hardware bind */
4. （可选）bm_sync_configure / arm
5. bm_exec_start_all()                    /* start 实例 → bm_hrt_start */
6. （可选）bm_sync_trigger
7. bm_ticker_init(slots)
8. bm_event_freeze_subscriptions()          /* 确定性 profile 建议 */
9. 主循环：
       bm_exec_drain_streams(budget)
     → bm_ticker_poll()
     → bm_event_process(budget)
     → bm_wdg_feed()
```

## 10. `bm_exec` 组装流程

```text
assemble_tables()
  对每个 slot：
    g_bindings[i] = { instance, slot }
    若 PERIODIC：
      g_hrt_slots[j] = { period_us, TIMER, bm_exec_run_binding, &g_bindings[i] }
bm_hrt_init(g_hrt_slots, ...)
各实例 ops->init()
对每个 HARDWARE slot：
  hal_binding = { bm_exec_run_binding, &g_bindings[i] }
  slot->bind(inst, &hal_binding)
```

| API | 作用 |
|-----|------|
| `bm_exec_init_all` | 校验、组表、`bm_hrt_init`、硬件 bind；会话 → INITED |
| `bm_exec_start_all` | 各实例 `ops->start`，全部成功后再 `bm_hrt_start`；会话 → STARTED |
| `bm_exec_safe_stop_all` | 关闭会话门、停 HRT、解绑 Hardware、逆序 safe_stop |

CMake/`bm_config.h` 见 [../02-构建与工具链/01-CMake选项与bm_config](../02-构建与工具链/01-CMake选项与bm_config.md)。
