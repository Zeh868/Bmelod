# 03 运行时约束与排障

> **本文职责**：fail-stop 策略、域隔离禁止项与排障表，供日常查阅。  
> **不负责**：单元测试操作步骤 → [01-单元测试与排障](01-单元测试与排障.md)。

本文说明框架在运行时的**行为边界与禁止事项**，不代表已通过 IEC 61508 / SIL 正式认证。

---

## 1. 故障策略：fail-stop

运行时发现以下情况时，返回确定性错误码（如 `BM_ERR_INVALID`、`BM_ERR_OVERFLOW`、`BM_ERR_NOT_INIT`），**不自动复位**控制块或掩盖故障：

- 索引越界、算术溢出
- 非法状态机转换
- 重复释放、双注册
- 启动校验失败

由应用或上层监督逻辑决定：重试、降级、安全停机。

---

## 2. 域隔离禁止项

| 规则 | 原因 | 违反时的典型现象 |
|------|------|------------------|
| HRT ISR 禁止 `bm_event_process`、订阅回调 | WCET 不可控 | 电流环抖动、偶发死锁 |
| HRT ISR 禁止 `bm_mempool_alloc`（常规路径）、bm_bus QUEUE 模式 | SRT 数据结构 | 数据撕裂、优先级反转 |
| HRT ISR 禁止 `BM_LOG` | 格式化耗时不可分析 | 错过 PWM 截止 |
| HRT 可用 `publish_copy_from_isr` | 仅入队，不执行回调 | 误以为 ISR 内回调已跑 |
| SRT 可读 `bm_bus` LATEST；HRT 可写 | 单写单读无锁契约 | 读者看到半更新数据 |
| `bm_shell` 仅调试构建 | 防未授权命令 | 生产固件误开 Shell |
| `start_all` 失败回滚 | 停 HRT、解绑硬件 | 半启动态下 IRQ 仍触发 |

### 2.1 优先级掩码模式（`BM_CONFIG_ENABLE_PRIORITY_MASK=1`）

- 低于 HRT 阈值的普通 ISR 调用 `*_from_isr` 变体仍被正确互斥。
- **HRT 级（≥ 阈值）ISR 不得**操作事件/ultra/内存池队列。
- 非掩码模式（默认）：`BM_CRITICAL_ENTER()` 全关中断，任意 ISR 与主循环互斥。

### 2.2 流式契约（已落地为代码守卫）

| 契约 | 说明 |
|------|------|
| `bm_stream` cache 失败 | `producer_commit` 失败时框架内部回收块并计 `stats.corrupt`；调用方**不应**再 `producer_abort` |
| `on_ready` 与 `exec_drain` 互斥 | 同一 stream 不得同时挂 `on_ready` 与 block/frame 槽 |

## 3. 看门狗与启动

| 规则 | 说明 |
|------|------|
| 软件看门狗模块窗口 | 未 `feed_module` 则不喂硬件狗 |
| 零模块看门狗 | 未注册软件模块时 `bm_wdg_feed` 直接喂硬件狗 |
| `bm_exec_init_all` 校验失败 | 不启动 HRT，返回错误码 |
| `bm_exec_start_all` 失败 | 停 HRT、解绑、已启动实例逆序停止 |

## 4. 排障速查表

### 4.1 事件与模块

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| `get_dropped_count` 递增 | 队列满 | 加大队列 / 提高 `process` budget |
| `subscribe` → `BM_ERR_BUSY` | 已 freeze | 仅在 freeze 前订阅 |
| 回调内 `process` → `BM_ERR_BUSY` | 非可重入 | 回调内只 `publish_copy` |
| 模块 init 失败 | 优先级或依赖错误 | 查 `bm_module_init_all` 回滚日志 |

### 4.2 混合域

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 电流环不跑 | 未 `bind_complete` 或未 `bm_hrt_start` | 查 `bm_exec_get_session` |
| `deadline_missed` 递增 | 环 WCET 超周期 | 减算法量或降频 |
| snapshot 读者数据跳变 | 写者过快 | 正常（最新值语义）；需全帧则改协议 |
| `sync trigger` 无效果 | 未 arm 或 HAL stub | 链 `bm_sync_hal_native` 或真机 HAL |
| stream 块积压 `mark_late` | drain budget 不足 | 先 drain 再 event；加大 budget |

## 5. 残余风险（使用前必读）

1. **HAL 弱符号**：未链接平台 HAL 时，外设调用可能静默失败 → 生产构建必须链接 `bm_hal_<platform>`。
2. **零拷贝事件**：`bm_event_publish_event` 生命周期由调用方保证。
3. **WCET**：内存池 `alloc` 清零在锁内完成（防撕裂）；最终上界须结合 MCU 与实测。
4. **测试注入钩子**：`BM_ENABLE_EVENT_TEST_HOOK` 等仅用于测试固件。
5. **`bm_mempool_reset`**：仅复位位图，不清理池内对象。

## 6. 验证命令

```bash
cmake -B build-sil2 -DBM_BUILD_TESTS=ON \
  -DBM_ENABLE_SHELL=OFF \
  -DBM_ENABLE_HRT=ON -DBM_ENABLE_TICKER=ON \
  -DBM_ENABLE_EXEC=ON -DBM_ENABLE_SYNC=ON \
  -DBM_SYNC_HAL_NATIVE=ON
cmake --build build-sil2 --config Debug
ctest --test-dir build-sil2 -C Debug --output-on-failure
```

更多测试说明见 [01-单元测试与排障](01-单元测试与排障.md)。

---

## 7. 建议的工程实践

1. 生产固件关闭 `BM_ENABLE_SHELL` 或剥离命令表。
2. CI 跑全量单元测试 + QEMU 冒烟。
3. 关键 HRT 回调避免动态行为；保持无分支深调用。
